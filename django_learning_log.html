El Projecte "Learning Log"

Creació d'una aplicació web interactiva amb Django on els usuaris poden registrar-se i guardar els seus aprenentatges. Aquest projecte avalua directament el Tractament de Dades (Saber Bàsic B).

Saber Bàsic B Framework Django

📖 Què és Django i per què l'utilitzem?

Fins ara hem programat codi que s'executa exclusivament a la nostra terminal negra. Pel terminal hem creat calculadores, jocs i llistes. Però les aplicacions reals viuen a internet i el món les utilitza a través d'un navegador web interactiu (com Chrome o Firefox).

Django és un Framework d'Aplicacions Web. Un framework és un conjunt d'eines i plantilles de codi que ens pre-construeixen les parts avorrides de fer una pàgina web (com on s'han de guardar les contrasenyes de manera segura o com connectar-se a una base de dades).

🎯 Relació amb el Saber Bàsic B (Tractament de Dades)

Desenvolupar un back-end amb Django no tracta tant sobre "dibuixar la web", sinó sobre gestionar informació massiva i relacional.

Utilitzarem el Model Entitat-Relació. Nosaltres definirem estructures en Python (anomenades Models) que Django traduirà automàticament a comandes SQL exactes per a crear taules, guardar registres d'usuaris i recuperar-los de la nostra Base de Dades.

🛠️ Pas 1. Preparant el Terreny

1. L'Entorn Virtual (`venv`)

Instal·lar eines directament a l'ordinador pot provocar que projectes antics deixin de funcionar si s'actualitzen. Django SEMPRE s'instal·la dins d'un espai aïllat anomenat Entorn Virtual.

Terminal (Windows / VS Code)
# 1. Crear la carpeta i entrar (a la consola)
mkdir learning_log
cd learning_log

# 2. Crear l'entorn virtual (anomenat 'll_env')
python -m venv ll_env

# 3. Activar l'entorn virtual
ll_env\Scripts\activate

2. Instal·lar i Arrancar Django

Amb el nostre entorn virtual encès (ho sabràs perquè et sortirà el nom de l'entorn virtual davant del cursor), procedirem a descarregar el Framework d'internet.

Terminal
# Instal·lar
pip install django

# Crear l'estructura base del projecte (atenció al punt final!)
django-admin startproject config .

# Crear la nostra primera BD i arrancar el servidor
python manage.py migrate
python manage.py runserver

🗄️ Pas 2. Definim el Model de Dades

Imaginem que volem permetre a l'usuari crear "Temes" (Topic) d'aprenentatge. Per exemple: "Escacs" o "Codis HTML". Django necessita saber què és un "Tema" perquè ho pugui guardar a la Base de Dades.

Saber Bàsic B: Aquesta classe Python que veus aquí avall és en realitat l'equivalent a fer un CREATE TABLE en llenguatge SQL.

models.py (A dins de la teva App)
from django.db import models

class Topic(models.Model):
    """Un tema sobre el qual l'usuari està aprenent."""
    
    text = models.CharField(max_length=200)
    date_added = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        """Retorna una representació en cadena del model."""
        return self.text

Per què és important aquest pas?

  • 🔍 CharField avisa a la base de dades que estem guardant text curt i li limita la memòria (200 caràcters per seguretat).
  • ⏱️ DateTimeField guardarà automàticament el dia i l'hora exacta en què es va escriure la connexió sense que haguem d'ajustar el rellotge amb codi.
