Fonaments Teòrics de Back-End

En aquesta secció teòrica explorarem què és un Web Framework, com processen la informació els servidors i quines eines utilitzem per a tractar dades complexes des d'Internet.

Saber Bàsic B: Dades Conceptes

🖥️ 1. Front-End vs Back-End

El Front-End (Aparellador de l'usuari)

Tot allò que veiem i toquem al navegador. La seva feina és presentar la informació de manera agradable (HTML per l'estructura, CSS per la pintura, i una mica de JavaScript per fer que els botons reaccionin).

Allò Visible

El Back-End (El cervell i la memòria)

El servidor amagat. És l'encarregat de validar els usuaris que entren, consultar el catàleg a la base de dades, realitzar càlculs complexos i enviar els paquets d'informació de tornada al navegador. Aquí és on conviu la programació robusta (C, Python) amb les dades (SQL).

L'Estructura Interna

📐 2. L'Arquitectura de Django (El patró MTV)

Quan estem construint plataformes gigantesques, no escrivim tot el codi al mateix fitxer. Els Frameworks (com Django per Python o Laravel per PHP) organitzen el projecte dividint-lo en tres parts clarament diferenciades. Aquest patró teòric s'anomena usualment MVC (Model, Vista, Controlador), però Django el simplifica a MTV:

  • [M] Model: La lògica de dades. És l'esquema de com seran les taules a la Base de Dades i les regles per emmagatzemar la informació.
  • [T] Template (Plantilla): La cara visible. L'esquelet HTML que rebrà les dades enviades pel codi i les posarà maques per a l'usuari final.
  • [V] View (Vista): El director d'orquestra. Un codi en Python que decideix quina dada treure del Model (SQL) i a quina Plantilla (HTML) enviar-la segons la ruta sol·licitada (ex. l'usuari obre 'domini.com/inici').

🗄️ 3. Què és un ORM? (Tractament d'Dades)

Un dels conceptes més potents en el món del desenvolupament de servidors és l'ORM (Object-Relational Mapping).

Tècnicament, les Bases de Dades només "parlen" el llenguatge SQL (ex. SELECT * FROM usuaris;). No obstant, nosaltres som programadors que treballem amb Classes i Objectes (POO). L'ORM fa de traductor simultani entre els dos mons.

🎯 L'Avantatge de l'ORM per a la Consistència de Dades

En lloc de construir estructures SQL manuals i arriscar-nos a cometre errors de tipatge o vulnerabilitats (com la Injecció d'SQL), l'alumne defineix una simple Classe a Python.
L'ORM del framework llegirà aquesta classe i es connectarà ell sol a la Base de Dades per construir totes les columnes amb els tipus de dades adients, assegurant la integritat del model.
